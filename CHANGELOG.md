# 更新日志

## 1.1.14

- 支持通过本机 PicGo Server 将 OmniDraw 的 `data:image` Base64 图片上传为公网 URL。
- 增加 PicGo 上传开关和 Server 地址配置，上传失败时不再错误使用默认人设图。

## 1.1.13

- 兼容仍会发送空工具参数的模型，自动从当前消息恢复视频动作提示词。
- 空参数不再触发 `missing prompt`，并增加恢复提示词的警告日志。

## 1.1.12

- 对齐 OmniDraw 的联动方式：要求先获取自拍工具返回的真实图片 URL，再传给 Seedance。
- 将 LLM Tool 的 `prompt` 设为必填，避免空参数调用导致提示词和首帧回退。
- 补充“自拍到视频”的工具调用顺序和首帧说明。

## 1.1.11

- 优化 AI 副脑提示词，默认使用单一连续场景、单一主要动作和轻微运镜。
- 参考图改为保留人物脸部、发型、体型和服装，只忽略背景，减少开头跳场景和人物动作僵硬。
- 增加 `motion_style` 配置，可选择 stable、normal 或 dynamic。

## 1.1.10

- 默认开启 AI 副脑扩写，更新后即可先生成剧本再调用 Seedance。

## 1.1.9

- 增加 AI 副脑：调用 AstrBot 当前文本模型扩写视频剧本。
- 支持自定义副脑系统提示词和超时时间。
- 副脑失败时自动回退到原始提示词。

## 1.1.8

- 增加高级提示词/剧本前缀配置。
- 增加“参考图仅用于人物”开关，隔离人物参考与背景环境。

## 1.1.7

- 默认严格使用配置页中的视频参数。
- 只有工具明确传入参数时才覆盖默认配置。
- 移除从自然语言强制解析时长和清晰度的逻辑。

## 1.1.6

- 当 LLM Tool 参数为空时，直接从原始消息解析时长、分辨率和音频开关。
- 增加消息参数解析日志，避免模型漏传参数时使用默认值。

## 1.1.5

- 详细记录最终提示词、参考图来源、参考图地址、模型和生成参数。
- 不记录 API Key。

## 1.1.4

- 修复视频回传分支的 Python 语法错误，恢复插件正常加载。

## 1.1.3

- LLM Tool 支持单独指定模型和是否生成音频。
- 增加视频体积优化说明。

## 1.1.2

- 修复图片提取顺序，确保检测到图片后使用 `image-to-video` 模式。
- 兼容 AstrBot `event.get_messages()` 图片链。
- 增加检测到的图片 URL 数量日志。

## 1.1.1

- 增加视频缓存保留天数配置，默认 3 天。
- 插件启动和生成任务前自动清理过期视频。

## 1.1.0

- 人设配置改为 AstrBot `template_list` 展开式界面。
- 增加人设参考图上传入口和中文字段说明。
- 兼容旧版扁平 profiles 配置。

## 1.0.9

- 修复 AstrBot 视频组件方法名，使用 `Video.fromFileSystem` 上传本地视频。

## 1.0.8

- 发送视频前先下载到本地，兼容 Kook 无法直接读取 Seedance CDN 视频地址的问题。
- 增加下载失败和本地上传日志。

## 1.0.7

- 增加任务提交、轮询、终态和视频回传的详细日志。

## 1.0.6

- LLM Tool 改为提交任务后立即返回，避免超过 AstrBot 240 秒工具超时。
- 视频完成后由插件后台主动发送结果。

## 1.0.5

- 增加当前激活人设和人设列表配置。
- 没有传图片时自动使用当前人设参考图。
- 自动把人设基础描述加入视频提示词。

## 1.0.4

- LLM Tool 参数为空时自动从当前消息提取视频提示词。
- 自动读取当前消息中的图片作为视频首帧。
- 增加更新日志文件。

## 1.0.3

- 修复 AstrBot 事件参数注入导致的序列化错误。
